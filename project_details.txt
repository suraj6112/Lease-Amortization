The current lease amortization process is highly manual, inconsistent, and not structured for automation, which makes it difficult to apply AI effectively.



1. Manual data extraction from contracts



Lease data is read manually from contracts and entered into a spreadsheet template. 
The amortization schedule is then created by copying and extending formulas.


Implication for AI:
There is no structured input layer—AI would first need to interpret unstructured contracts before anything else.



2. Fragmented and inconsistent file structure



Files are stored across SharePoint folders by company and clinic, but:
Some spreadsheets have one clinic per file
Others combine multiple clinics in one sheet
There is no consistent format across all schedules. 


Implication for AI:
Lack of standardization makes it hard to build a repeatable or scalable automation solution.



3. Repetitive, manual monthly processing



Each month, the process requires:
Manually shifting schedule data
Recalculating balances
Copying values into a journal entry file
This is repeated across dozens of schedules (60–70).


Implication for AI:
The problem is not just creation—it’s ongoing operational workload that is repetitive but not systematized.



4. Disconnected amortization and journal entry workflows



After updating schedules:
Key values are manually copied into a separate journal entry file
Entries are created and uploaded location by location


Implication for AI:
The process is split across multiple artifacts, rather than a single integrated flow.



5. Semi-standard template, but not truly system-driven



There is a “template,” but:
It still requires manual interpretation of each contract
It relies heavily on human judgment and setup


Implication for AI:
The template is not parameterized or driven by structured inputs, limiting automation potential.



Bottom-line Problem Statement



The lease amortization process today is:



Manual in data intake (contract → spreadsheet)
Inconsistent in structure (files, formats, layouts)
Repetitive in execution (monthly updates and journal entries)
Disconnected across steps (schedule → journal entry → upload)


As a result, it is not “AI-ready”—automation requires:



Structured inputs
Standardized schema
Formula-driven or system-driven outputs
